## How It Works

Each page now:

1. **Fetches Yoast SEO data** from WordPress GraphQL API using the `GET_PAGE_BY_SLUG` query
2. **Exports `generateMetadata()`** function that converts Yoast data to Next.js meta tags
3. **Renders `<JsonLdScript>`** component to inject JSON-LD schema for search engines

## Step-by-Step: What You Need to Do in WordPress

### 1. Edit Each Page/Post in WordPress

Go to WordPress admin → Pages/Posts → Edit the page

### 2. Scroll to Yoast SEO Box (at bottom)

Fill in the Yoast SEO fields:

```
📋 SEO Title:
   Your page title (shown in Google search results)
   Character: 30-60 recommended

📝 Slug:
   home, about, services, contact, portfolio, testimonials
   (Must match the WordPress page slug)

📄 Meta Description:
   Brief description of page (shown under title in Google)
   Character: 120-160 recommended

🖼️ OpenGraph Section:
   - OG Title: How page looks on Facebook/Social
   - OG Description: Social media description
   - OG Image: Image shown when shared

🐦 Twitter Section:
   - Twitter Title: How it looks on Twitter
   - Twitter Description: Description on Twitter
   - Twitter Image: Image for Twitter

🔗 Canonical URL:
   Usually auto-filled, but verify it's correct

📊 Schema:
   Advanced - usually auto-generated by Yoast
```

### 3. Click "Save Changes"

### 4. Update Your Next.js Frontend Domain

In each page file (app/page.tsx, app/about/page.tsx, etc.), change:

```typescript
url: "https://yourdomain.com/about",  // ← Change to your domain
```

## Troubleshooting

### ❌ Meta tags not showing?

**Check 1:** Is `generateMetadata` exported?

```typescript
export async function generateMetadata(): Promise<Metadata> {
  // Function body
}
```

**Check 2:** Are you visiting the correct URL?

- Homepage: `/` not `/index`
- Services: `/services` not `/service`

**Check 3:** Check browser console for errors

- F12 → Console tab
- Look for red error messages
- Check Network tab for GraphQL errors

### ❌ Wrong Yoast data showing?

**Check 1:** Is the slug correct in code?

```typescript
(GET_PAGE_BY_SLUG, { slug: "about" }); // Must match WordPress slug
```

**Check 2:** Is the WordPress page published?

- Go to WordPress → Pages → Check status is "Published"

**Check 3:** Has Yoast SEO been configured?

- Edit page in WordPress
- Scroll to Yoast SEO box
- Fill in: SEO Title, Meta Description
- Save

### ❌ JSON-LD schema not appearing?

**Check 1:** Is `JsonLdScript` component in JSX?

```typescript
return (
  <>
    <JsonLdScript yoastSeo={yoastData} />  // ← Must be here
    <div>Content...</div>
  </>
)
```

**Check 2:** Does Yoast have schema data?

- WordPress → Pages → Edit page
- Scroll to Yoast SEO
- Check "Advanced" section has schema

**Check 3:** Check browser console

- F12 → Console
- Look for JSON parsing errors

### ❌ Getting GraphQL errors?

**Check 1:** Verify WordPress GraphQL endpoint

```bash
# Test in browser
https://bluereeftech.com/demo-next-js/graphql
```

**Check 2:** Check environment variable

```bash
# .env.local should have:
NEXT_PUBLIC_WORDPRESS_API_URL=https://bluereeftech.com/demo-next-js/graphql
```

**Check 3:** Test query manually

```graphql
query GetPageBySlug {
  pageBy(slug: "about") {
    title
    slug
    yoastSeo {
      metaDesc
      opengraphTitle
    }
  }
}
```
